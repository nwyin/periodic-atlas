# atlas

Periodic-element supply-chain atlas. Source of truth is the per-element YAML files under `elements/`; everything else (parquet, DuckDB, static HTML viewer) is generated from them.

## Prerequisites

- Python 3.12+ via [`uv`](https://github.com/astral-sh/uv)

First-time setup:

```sh
cd atlas
uv sync
```

## Re-building the viewer

After editing any `elements/*.yaml` file, run these three steps from the `atlas/` directory:

```sh
uv run scripts/validate.py       # pydantic-validate all 105 element files
uv run scripts/build.py          # write parquet tables + atlas-2025.duckdb to build/
uv run scripts/build_viewer.py   # regenerate static HTML in viewer/
```

The viewer then lives at `viewer/index.html`.

### One-liner

```sh
uv run scripts/validate.py && uv run scripts/build.py && uv run scripts/build_viewer.py
```

If `validate.py` fails the pipeline stops; fix the YAML and re-run.

## Serving the viewer locally

The viewer is static HTML + JS, so any static file server works. Easiest is Python's built-in one — from the `atlas/` directory:

```sh
uv run python -m http.server 8000 --directory viewer
```

Then open <http://localhost:8000/> in a browser.

If you'd rather run the server from inside `viewer/`:

```sh
cd viewer && uv run python -m http.server 8000
```

To use a different port, pass it as the last argument (e.g. `8080`). Press Ctrl+C to stop the server.

> Some browsers block the JS chart fetches against `file://` URLs, so opening `viewer/index.html` directly can render a blank page — running the static server is the reliable path.

## Optional: verification summary

```sh
uv run scripts/verify_report.py   # aggregate verification/*.md into build/verification_summary.json
```

## Layout

```
atlas/
├── elements/                # source YAMLs, one per element (Au.yaml, Cu.yaml, …)
├── verification/            # per-element claim-verification notes (Au.md, …)
├── scripts/
│   ├── validate.py          # pydantic validation over all elements
│   ├── build.py             # YAML → parquet + duckdb
│   ├── build_viewer.py      # duckdb → static HTML
│   └── verify_report.py     # verification/*.md → build/verification_summary.json
├── src/atlas/models.py      # pydantic schema (v0.2)
├── viewer/                  # generated HTML + JS charts (regenerated by build_viewer.py)
├── build/                   # generated parquet + duckdb artefacts (gitignored)
├── tests/                   # pytest suite for build + build_viewer
├── zz-decomposition-plan.md # methodology note for USGS "Other countries" decomposition
└── pyproject.toml
```

## Tests

```sh
uv run pytest
```

## Conventions

- `ruff check` / `ruff format` with `line-length = 144` (see `pyproject.toml`).
- Every factual claim in a YAML file must carry a `source_id` resolving to an entry in the file's `sources` list. Validation enforces this.
- `ZZ` is the ISO-style code used for the "rest of world" residual in country share lists — see `zz-decomposition-plan.md` for the secondary-source augmentation methodology.
